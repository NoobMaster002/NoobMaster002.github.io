# 一

1.url 解析：判断输入的内容是关键字还是 url 地址，如果是 url 地址的话会进行 dns 解析，查找该 url 的 ip 地址、dns 查找过程：
①去浏览器缓存中查找，浏览器会缓存 dns 记录一段时间，但是系统并没有告诉浏览器要缓存多长时间，不同的浏览器缓存时间不同，一般为 2-30 分钟
② 如果浏览器缓存中查不到的话浏览器会进行一个系统调用去系统缓存中查找，之后可能要查找路由器缓存和网络服务提供商 isp 缓存 dns 的服务器，在这一般都可以找到缓存记录，还有一个就是递归搜索：你的网络服务提供商会从根域名服务器开始进行递归搜索，从顶级域名服务器到 url 的域名服务器

# 二

2. 找到 ip 地址之后浏览器会建立 tcp 链接，进行 tcp 三次握手：
第一次客户端向服务端发送 syn 包（同步序列编号）客户端进入 syn_sent 状态，
第二次服务端收到并确认客户端发送的 syn，同时服务端向客户端发送 syn+ack，服务端进入 syn——recover 状态，
第三次客户端收到服务端发送的 syn+ack，同时向服务端发送 ack 确认字段，服务端和客户端进入 established 状态，tcp 连接成功。
客户端给服务器发送一个 http 请求，请求头包括请求方法，http 协议版本 url 地址，请求行有 accept 接受什么类型的相应，connection（keep-alive 保持 tcp 链接）之类的，请求空行，请求体，期间可能会有重定向，

# 三

3. 服务端处理请求，响应头包括 http 协议版本，响应状态码，相应行 Pragma，cache-control（缓存）相应空行 响应体

# 四

4.tcp 四次挥手：
第一次 客户端发送连接释放报文段，停止发送数据，主动关闭 tcp 链接，进入 fin wait 1 状态 等待服务端的响应
第二次 服务端收到连接释放报文段后发出确认报文段，服务端进入 close wait 状态，此时 tcp 处于半关闭状态，客户端收到服务端的确认后进入 fin wait 2 状态，等待服务端发出的连接释放报文段
第三次 服务端数据发送完毕后发出连接释放报文段，服务端进入 last -ack 状态，等待客户端的确认
第四次 客户端收到服务端的连接释放报文段后发出确认报文段，客户端进入 time wait 状态，此时 tcp 链接未释放掉，需要等待最大报文生存时间结束后客户端进入 close 状态

# 五

5. 浏览器渲染页面，浏览器渲染的顺序：
1. 构建 dom 树
2. 构建 css 树
3. 构建渲染树
4. 节点布局
5. 页面渲染
